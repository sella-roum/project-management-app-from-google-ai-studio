# Plan

## Objective
- モバイルアプリのライトテーマ化に向けて現状分析を行い、実装計画を明文化する。
- UI/UX を世界最高峰水準に近づけるための改善点を洗い出し、記録する。
- 分析結果を元に、実装のためのタスクリストを作成する。
- 実装タスクリストに沿ってライトテーマ統一とUI/UX改善を実施する。

## Scope
- In:
  - モバイルアプリ（`apps/mobile`）のテーマ設計・カラー利用状況の分析
  - ライトテーマ統一の実装計画作成
  - UI/UX 改善点の分析と整理
  - 実装タスクリストの作成
  - 実装タスクの実行（ライトテーマ統一/UX改善/品質チェック）
- Out:
  - 実際の実装変更（本タスクでは計画のみ）

## Assumptions
- 既存のテーマトークン（`Colors.light`/`Colors.dark`）を基盤として統一する方針とする。
- 実装段階で UI 変更が大きい場合は別途確認が必要になる。

## Approach
- `apps/mobile/constants/theme.ts` と `useThemeColor` の利用箇所を確認し、ダーク基準の要因を整理。
- ハードコード色の存在箇所を洗い出し、ライトテーマ統一の方針を立案。
- 計画を `.codex/runs/<run_id>/ANALYSIS.md` に記載。

## Definition of Done
- 現状分析と実装計画が Markdown で記載されている。
- `docs/PROJECT_CONTEXT.md` にテーマ周りの理解が追記されている。

## Risks / Unknowns
- 画面ごとの固定色が多い場合、移行作業の工数が増える可能性。
- デザイン仕様が未定義のため、配色変更の基準が不明確な箇所がある可能性。
