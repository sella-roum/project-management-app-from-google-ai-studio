# Mobile UI/UX ブラッシュアップ計画（Apple級の洗練を目指す）

## 目的
- Web を仕様の一次情報として、**機能は同等のまま**モバイル体験を「驚くほど洗練された」品質に引き上げる。
- Apple/YC/F500のプロダクト水準を常にベンチマークし、**情報設計・操作導線・視覚品質**を徹底的に磨く。

## 現状の観察（モバイル実装の特徴）
- **タイポグラフィ階層が浅い**：`ThemedText` で `default/title/subtitle` など基本のみ。見出し・本文・キャプションの差が弱く、情報の強弱が出にくい。
- **カラーパレットが限定的**：`theme.ts` は tint と数色のみで、**状態色/ブランドカラー/表面色**の体系が不足。
- **カード/バッジの視覚的洗練が不足**：`issue-card.tsx` などは機能的だが、影/グラデ/階層が弱く「プロダクト感」が出にくい。
- **ホームの情報密度は適切だが、視覚的なリズムが単調**：カード/統計/リンクが同一トーンで並び、ヒーロー要素やコンテキストが希薄。
- **空状態/ロード/エラーが体験として弱い**：スケルトンや空状態のトーンが均一で、ブランド体験になっていない。
- **操作の可視性が弱い**：タップ可能領域や主要CTAの強弱が薄く、迷いが生まれやすい。
- **情報のグルーピングが不足**：統計・タスク・最近見たの意味的境界が視覚的に弱い。
- **操作時のリアクションが控えめ**：Haptics/アニメーション/状態変化が機能的で止まっている。
- **色の階調と明度差が単調**：Surface/Background/Border の階調差が小さく、深度が出にくい。
- **ブランドカラーの活用範囲が限定的**：主要CTA以外でのアクセント活用が少ない。

## ブラッシュアップのゴール（体験指標）
1. **視線誘導が自然に起きる**：重要情報→次アクションへ迷いなく移動できる。
2. **ブランドの品格**：余白・影・角・色の使い方が一貫し、触った瞬間に「高品質」と感じる。
3. **操作の気持ちよさ**：タップ時の反応、遷移の動き、状態変化が一瞬で理解できる。
4. **“迷わない”情報設計**：どの画面でも次の一手が明確に見える。
5. **一貫性のあるディテール**：カード/ボタン/フォームの挙動が全画面で統一される。
6. **色の安心感と視認性**：疲れにくく、重要度が即理解できる配色設計。

## 実行プラン（段階別）

### フェーズ1：UX監査 & 仕様整合
- Web を仕様の一次情報として、**導線・情報設計・操作結果をモバイルに完全一致**させる。
- 主要導線の棚卸し（Home / Projects / Issue / Search / Notifications / Profile）。
- **UIは変えてよいが、機能差を作らない**前提で改善対象を整理。
- `docs/feature_parity.md` を確認し、差分がある場合は理由と対応計画を明記。
- KPI 観点の整理（タスク完了率・検索→詳細遷移率・通知既読率など）を用意し、UX改善の指標化を行う。

### フェーズ2：デザインシステム再構築
- **色の体系化**
  - Brand / Surface / Text / State / Accent を明確化。
  - Light/Dark の視認性とコントラスト基準を設計。
  - Surfaceレイヤーを3段（base/raised/overlay）で定義し、カード/シート/モーダルに適用。
  - 緊急度（赤/橙/黄/緑）のトーンを淡色・濃色のペアで設計し、バッジ/背景/テキストで使い分け。
- **トーン&モード定義**
  - 「集中モード」「レビュー」「閲覧」のトーンを色と密度で差別化。
  - 仕事モードに適した、目に優しい中間階調を整備。
- **タイポグラフィ再設計**
  - `Display / Title / Headline / Body / Caption` の5段階。
  - Apple系の密度と読み心地に寄せる。
- **空間設計**
  - 8pt grid を軸に、カード/セクションの余白を再配置。
  - 間の設計で「上質さ」を作る。
- **影/境界/角丸の規律化**
  - Elevationを3〜4段に統一し、階層感を固定化。
  - 角丸サイズのルールを定義し、カード/ボタンに適用。
  - ボーダー色は同系統のニュートラルで明度差を付け、情報の区切りを視覚化。

### フェーズ3：コンポーネント刷新
- **カード系（Issue/Project/Stat）**
  - 影・境界線・背景色で階層を明確化。
  - 状態バッジを洗練されたチップに統合。
- **CTA/ボタン**
  - Primary / Secondary / Ghost を再定義。
  - 触覚フィードバックとアニメーション強化。
  - Primaryはブランドカラーを強く、Secondaryはニュートラルベースで可読性を担保。
- **ナビゲーション/ヘッダー**
  - タブ/トップバーの視認性と統一感を改善。
  - 重要なアクション導線を迷わせない設計へ。
- **フォーム/入力**
  - ラベルの視認性、フォーカス時の強調、入力補助を統一。
  - エラー/警告/成功の視覚言語を整理。
- **リスト/バッジ**
  - 状態チップ、優先度表示、期限バッジを統一パターン化。
  - 情報密度に応じた2段/1段のレイアウト切り替え。
  - 色は「状態」「優先度」「期限切れ」で意味を固定し、意味色の乱用を禁止。

### フェーズ4：画面ごとのUX強化
- **Home**
  - 1stビューに「今日の目的」と要約を明確化。
  - 統計カードをダッシュボード化し視覚的なリズムを追加。
  - 視線誘導のため、ヒーロー領域にブランドアクセントを軽く配置。
- **Issue 詳細**
  - 重要情報（Status/Priority/Due/Assignee）を最上部に集約し、一目で理解できる構成へ。
  - コメント/履歴の情報密度を調整し、読みやすい視認階層を付与。
- **Projects / Issue**
  - 状態・期限・優先度の視認性を改善。
  - 空状態/ロード状態をブランド体験として設計。
- **Search / Notifications**
  - 検索結果の並び替えやフィルタの視覚整理。
  - フィルタチップに状態色を採用し、意味が即理解できる色設計に。
- **Profile**
  - 統計/アクション/設定の情報階層を整理し、余白と視線誘導を最適化。
- **Onboarding / Setup**
  - 初回体験のトーン設計（安心感・期待感）を強化。

### フェーズ5：モーション & マイクロインタラクション
- **アニメーションは速く、短く、気持ちいい**
  - タップ時のスケール/ハプティクス。
  - リスト追加・状態更新時の自然な遷移。
- **ステータス変化に即時性を付与**
  - 変化が一目でわかるトランジション設計。
- **空状態/ロードの演出**
  - スケルトンや空状態にブランドの“らしさ”を入れる。
  - 体験の待ち時間を「上質な演出」として感じさせる。

### フェーズ6：品質ゲートと体験確認
- `lint / typecheck` は毎回通す。
- Web / Mobile で同じユーザーストーリーを再現し、**結果が一致**していることを確認。
- 起動確認（Expo + Web）で UI を実際に触り、必要なら微調整。
- 実機での視認性（屋外・暗所）チェックを行い、コントラストと文字サイズを再調整。
- 操作の到達コスト（タップ数・スクロール量）をレビューし、効率化を図る。

## 成果物の定義
- **視覚デザイン**：Apple級の余白・フォント・色設計。
- **操作体験**：押したくなるボタン・読みやすい情報階層。
- **Parity遵守**：Webとの機能差ゼロ。
- **定量/定性の改善**：ユーザビリティテストでの完了率向上と、主観評価の向上。

## 期待する最終体験
「このアプリは世界基準のデザインだ」と感じるだけでなく、
**“触っている時間そのものが心地よい”** 体験を提供すること。
さらに、**「迷わない・疲れない・期待を超える」** を全画面で実現する。
