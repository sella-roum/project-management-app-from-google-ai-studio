# Mobile UX デザインレビュー判定基準

## 採点観点（5段階）
1. **Before/After差分の明瞭さ**
   - 1: 差分がわかりにくい
   - 5: 主要要素の改善が一目で分かる
2. **可読性**
   - 1: 文字が小さい/読みにくい
   - 5: 見出し/本文/補足の階層が明確
3. **コントラスト**
   - 1: 背景と文字の区別が困難
   - 5: WCAG AA相当を満たし視認性が高い
4. **操作回数/導線**
   - 1: 主要操作までの手数が多い
   - 5: 主要操作が1〜2手で到達
5. **一貫性（トークン/コンポーネント）**
   - 1: 画面ごとに見た目がバラバラ
   - 5: 余白/角丸/色/ボタンが統一

## レビュー用チェックリスト
- [ ] 画面ごとの主要CTAがファーストビューに存在する
- [ ] 状態/優先度/期限の表示ルールが統一されている
- [ ] フォーム入力のフォーカス/エラー表現が一致している
- [ ] 8pt grid に沿った余白調整が反映されている
- [ ] Light/Darkで同等の可読性がある
